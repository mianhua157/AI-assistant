# 项目日志

## 2026-04-20 - RAG 检索优化

### 问题背景
当候选池里只有 1 条 wiki 时，检索效果不佳。需要增加 wiki 在向量库中的数量。

### 修改内容

#### 1. rag.py - 扩大候选池
- 将 `similarity_search` 的 k 从 8 改成 15
- 更容易从更大的候选池里捞到足够的 wiki 文档
- 保持 `docs = wiki_docs[:2] + raw_docs[:3]` 的组合逻辑

#### 2. build_vectorstore.py - Wiki 文档切分
- 新增 `split_wiki_documents()` 函数
- 使用 RecursiveCharacterTextSplitter 切分 wiki 文档
- 参数：chunk_size=500, chunk_overlap=100
- 在 `load_all_documents()` 中调用切分函数
- 目的：增加 wiki 在向量库中的 chunk 数量，提高被检索到的概率

#### 3. build_wiki.py - 优化生成 Prompt
- 强制 wiki 内容结构，让 embedding 更"像问题"
- 必须包含的 5 个部分：
  1. 定义（最重要）- 用 1-2 句话清晰定义该概念
  2. 训练过程
  3. 预测过程
  4. 常见方法
  5. 与其他概念的关系
- 避免生成"资料中未充分覆盖该主题"这样的无效内容

#### 4. build_wiki.py - 通用 PDF 加载
- 新增 `load_pdf_documents()` 函数
- 遍历 `raw/` 目录读取所有 PDF 文件
- 修改 `main()` 函数，从 `raw/` 目录加载所有 PDF
- 不再硬编码单个 PDF 文件名

### 预期效果
- 向量库中 wiki chunk 数量增加
- 检索时更容易命中 wiki 文档
- 回答质量提升（wiki 内容更结构化）
- build_wiki.py 可以自动读取 raw/ 下所有 PDF，更通用

## 2026-04-21 - Wiki 优先 + 限制数量

### 修改内容

#### rag.py - 文档选择逻辑优化

**版本 1：wiki 优先 + 限制数量**
```python
selected_docs = wiki_docs[:2]
if len(selected_docs) < 2:
    selected_docs += raw_docs[:1]
```

**版本 2：固定"1 wiki + 1 raw + 补充"**
```python
docs = []
if wiki_docs:
    docs.append(wiki_docs[0])
if raw_docs:
    docs.append(raw_docs[0])
remaining = wiki_docs[1:] + raw_docs[1:]
docs.extend(remaining[:2])
```

**版本 3：每个 doc 限制 600 字符**
```python
context = "\n\n".join([doc.page_content[:600] for doc in docs])
```

### 预期效果
- 优先使用 wiki 文档
- 保证 raw 也有代表
- 减少 token 消耗（最多 4 个文档，每个 600 字符）

## 2026-04-21 - 比较类问题优化

### 修改内容

#### rag.py - 智能 k 值 + 文档选择

```python
# 检测问题类型
is_definition = q.startswith("what is") or q.startswith("什么是") or q.startswith("定义")
is_compare = "compare" in q or "对比" in q or "比较" in q or "vs" in q or "difference" in q

# 根据类型调整 k 值
if is_definition:
    candidates = vectorstore.similarity_search(rewritten_query, k=3)
elif is_compare:
    candidates = vectorstore.similarity_search(rewritten_query, k=8)  # 更多候选
else:
    candidates = vectorstore.similarity_search(rewritten_query, k=6)

# 根据类型选择 wiki 数量
if is_compare:
    docs.extend(wiki_docs[:2])  # 比较类拿 2 个 wiki
else:
    if wiki_docs:
        docs.append(wiki_docs[0])  # 定义类只拿 1 个
```

#### rag.py - Prompt 差异化
- 比较类：`"比较类问题请列出两者的异同点（可以用表格或分点）"`
- 定义类：`"只给核心定义，不要展开推导"`

### 预期效果
| 问题类型 | 示例 | k 值 | wiki 数量 |
|----------|------|-----|-----------|
| 定义类 | `What is classification?` | 3 | 1 |
| 比较类 | `Compare classification and regression` | 8 | 2 |
| 其他 | `解释一下 SVM` | 6 | 1 |

- 比较类问题有 2 个 wiki 对比
- 定义类问题更精准（不混入无关 wiki）
- Prompt 根据类型调整回答策略
